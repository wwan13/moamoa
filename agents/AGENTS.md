# 기본 AGENT 규칙

이 문서는 moamoa 프로젝트에서 **어떤 작업을 수행하든 공통으로 적용되는 작업 원칙**을 정의한다.

## 작업 원칙
- 요청된 범위 내에서만 작업한다.
- 목표 달성을 위한 **최소 변경**을 우선한다.
- 명시적으로 요청되지 않은 기능 추가는 하지 않는다.
- 기존 코드 스타일과 패턴을 우선적으로 따른다.

## 안전 규칙
- 요청과 무관한 코드 리팩토링을 하지 않는다.
- 공용 유틸, 공통 설정, 인프라 설정은 요청이 없는 한 수정하지 않는다.
- 퍼블릭 API 변경은 요청이 없는 한 금지한다.
- 대규모 구조 변경이 필요하다고 판단되면 먼저 가정을 명시한다.

## 가정 처리
- 요구사항이 불명확한 경우, **가정을 명확히 서술한 뒤 작업을 진행**한다.
- 추측으로 동작을 변경하지 않는다.

## 출력 규칙
- 변경 요약을 먼저 제공한다.
- 변경된 파일 목록만 나열한다.
- 코드 출력은 **실제로 변경된 파일에 한해서만** 제공한다.
