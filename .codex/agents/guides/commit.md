# Commit

이 문서는 moamoa 저장소의 커밋 작성 기준이다.

## 기본 원칙
- 한 커밋은 한 가지 목적만 담는다.
- 리팩토링/기능/버그수정은 가능한 분리한다.
- 동작 변경이 없으면 제목에 `refactor`를 사용한다.

## 메시지 형식
`<type>: <summary>`

예시 타입:
- `feat`: 기능 추가
- `fix`: 버그 수정
- `refactor`: 동작 변경 없는 구조 개선
- `test`: 테스트 추가/수정
- `docs`: 문서 변경
- `chore`: 빌드/의존성/설정

## 작성 규칙
- summary는 50자 내외로 짧고 구체적으로 작성한다.
- 무엇을 왜 바꿨는지 본문에 1~3줄로 남긴다.
- 파일 목록 나열 대신 영향 범위와 의도를 적는다.

## 금지
- 의미 없는 메시지(`update`, `fix bug`, `wip`) 금지
- 서로 무관한 변경을 한 커밋에 혼합 금지
